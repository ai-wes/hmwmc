Homeostatic Predictive Memory (HPM)

A surprise-gated working memory that decides when to remember based on how unexpected an event is, relative to what each memory slot has learned to expect.

1 Input: Hidden State Sequence

HPM receives the hidden state sequence from the main world model backbone — that's the output of the 6 transformer-style blocks after episodic memory integration.


The input is h_seq with shape (B, T, D) where B is batch size, T is episode length (24–64 steps depending on tier), and D is the model dimension (256).


At each timestep t, HPM sees h_t — the model's current belief about the world state. HPM's job is to decide: "Is this moment worth remembering?"


Crucially, HPM operates separately from the main sequence. It doesn't modify the backbone's output — it produces a parallel memory track that the QueryHead reads later. This separation means next-step prediction keeps working normally while HPM learns what to retain for answering questions.

2 Surprise Computation

This is the core insight of HPM: memory writes are gated by how surprising an event is, relative to what each slot has learned to expect — not by raw prediction error magnitude.


Each of the 4 slots has its own one-step-ahead predictor — a per-slot linear layer (SlotLinear) that tries to predict h_t from h_{t-1}. The prediction error is: e_t = mean( (pred − h_t)² )


Raw error alone isn't useful — a freshly initialized model has high error everywhere. So HPM z-scores the error against running statistics:


z_t = (e_t − μ) / σ


The running mean μ and standard deviation σ are tracked per-slot via EMA (decay=0.99). This means a mature model with low σ becomes more sensitive to rare events, while a young model with high σ writes conservatively. Signal-to-noise is handled by construction.


All of this is detached from gradients. The surprise signal is purely a control mechanism — it tells the gate how much to open, but the learning signal flows through the gate and write path, not through the surprise computation itself.

3 Gate: How Much to Write

The gate decides how much of the current hidden state to write into each memory slot. A gate value near 1.0 means "overwrite the slot with new content." A gate near 0.0 means "keep what's already stored."


The gate MLP takes three inputs concatenated together:


• h_t — what's happening right now (256 dims)
• z_t — how surprising it is (1 dim per slot)
• state_embed — the slot's current state as a learned embedding (8 dims)


These pass through a two-layer MLP (fc1 with GELU → fc2 → sigmoid) that is independent per slot via SlotLinear — each slot has its own weights, so they can learn different gating policies.


The raw gate g_raw is then scaled by a state-dependent gain:


• OPEN slots: gain = 1.0 (full plasticity)
• CLOSING slots: gain = 0.5 (reducing plasticity)
• LOCKED slots: gain = 0.1 (nearly frozen — preserving content)


This is where gradient flows. Unlike the surprise computation, the gate MLP is fully differentiable. The downstream loss (from query answers) teaches the gate which moments are worth remembering.

4 State Machine & Force Unlock

Each memory slot has its own three-state machine that controls long-term plasticity. This is directly inspired by the PNN (Progressive Neural Network) regulator elsewhere in the codebase, but applied to per-timestep memory writes instead of per-generation synaptic plasticity.


The state transitions are driven by gate_ema — an exponential moving average of the gate values. When a slot consistently writes little, it drifts toward LOCKED:


OPEN — Slot is actively updating. Full gate gain. If gate_ema drops below 0.10, transition to CLOSING.


CLOSING — Slot is settling. Gate gain halved to 0.5. If gate_ema drops below 0.03, transition to LOCKED. If it rises above 0.25, reopen.


LOCKED — Slot is preserving content. Gate gain = 0.1. Normal events barely overwrite. But if |z_t| > 3.0 (a truly extraordinary surprise), the slot is force-unlocked back to OPEN and gate is boosted to ~1.0.


The force-unlock is the emergency override. It means a locked slot that committed to tracking, say, a handoff chain will break open if something extremely unexpected happens — like a false-cue correction or a surprise chain-2 event. The first 50 steps (warmup) suppress force-unlocks while σ stabilizes.


The hypothesis is that this mechanism could lead to unsupervised slot specialization: different slots naturally locking onto different event types, then unlocking only for their "specialty" surprises.

5 Slot Memory & Write

HPM has 4 independent memory slots, each storing a (B, slot_dim) vector. The write update is a simple leaky recurrence:


w_t = (1 − g_t) · w_{t-1} + g_t · write_enc(h_t)


When the gate g_t is near 0 (not surprising), the old memory w_{t-1} passes through almost unchanged. When g_t is near 1 (very surprising), the slot is overwritten with fresh content from the write encoder.


Each slot has its own write encoder — another SlotLinear(D → slot_dim) — so different slots can learn to encode different aspects of the hidden state.


This recurrence is the mechanism by which gradient flows back in time. If the model needs to answer "who held the token 20 steps ago?", the gradient from the query loss travels backward through the chain of (1-g) multiplications until it reaches the step where g was high (the surprise event), and strengthens the gate and write encoder at that original moment.


With optional competitive=True, gate logits pass through softmax across slots before sigmoid — creating winner-take-all dynamics where the most surprised slot captures the write, and others are suppressed. This is the mechanism that could drive unsupervised slot specialization.

6 Read & Output

After running through all T timesteps, HPM has a slot sequence of shape (B, T, 4, slot_dim) — what each of the 4 slots remembered at each moment in time.


The read mode aggregates across slots to produce a single output per timestep:


• concat (default): Simply concatenate all 4 slot vectors. Output dim = 4 × 256 = 1024. Preserves slot identity — the QueryHead can learn which slot carries which information.


• mean: Average across slots. Output dim = 256. Loses slot identity but is more compact.


• attn: Query-conditioned attention — a learned projection of h_t attends over the slot vectors, so the read dynamically picks the most relevant slot per timestep. Output dim = 256.


The output hpm_seq is concatenated alongside the backbone's main sequence output and the holder features before feeding into the QueryHead. The QueryHead's input is: [sequence (256d) ; holder_features (max_entities) ; hpm_seq (1024d)].


HPM never modifies the main sequence — prediction heads (next-step, per-modality) operate on the unaltered backbone output. HPM's contribution is exclusively read by the query answering system.

∞ The Full Cycle — What Happens at Each Timestep

For every timestep t from 0 to T, HPM runs this cycle:


1. Predict: Each slot's predictor guesses what h_t will be, based on h_{t-1}. At t=0 there's no prior, so surprise is zero.


2. Measure surprise: Compare prediction to reality. Raw error is z-scored against the slot's running statistics. A z-score of 3+ means "this is a 3-sigma event for this particular slot" — very rare relative to what it's learned to expect.


3. Compute gate: The gate MLP combines the current hidden state, the surprise signal, and the slot's state embedding to produce a write probability. The state machine scales this by the slot's current plasticity.


4. Force-unlock check: If any slot sees |z| > 3.0 and we're past warmup, that slot is emergency-reopened regardless of its state.


5. Write: Each slot updates via the leaky recurrence. High gate means overwrite with new content. Low gate means keep the old memory.


6. Advance state machine: After the full sequence, update each slot's OPEN/CLOSING/LOCKED state based on its average gate activity.


After all T steps: Read out the slot contents, aggregate them, and hand the hpm_seq to the QueryHead. The backbone's main output is untouched — prediction heads never see HPM's memory.


The key insight: plasticity is governed by second-order statistics of prediction (surprise relative to expected surprise), not first-order error. A fresh model has high σ and writes conservatively. A mature model has low σ and is naturally sensitive to rare events. The memory bootstraps off the prediction mechanism — signal-to-noise is handled by construction.
